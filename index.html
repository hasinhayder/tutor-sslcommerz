Payment Flow

How Payments Work

Understanding the complete transaction lifecycle

Student Clicks "Purchase"

Student selects a course and initiates the checkout process from your Tutor LMS site.

Payment Request Sent

Plugin sends payment request to SSLCommerz API endpoint: /gwprocess/v4/api.php

Redirect to SSLCommerz

Student is redirected to the secure SSLCommerz payment page to complete the transaction.

Student Completes Payment

Student selects preferred payment method and completes the payment on SSLCommerz's secure page.

IPN Notification Received

SSLCommerz sends Instant Payment Notification (IPN) to your configured webhook URL.

Transaction Validation

Plugin validates transaction via SSLCommerz API: /validator/api/validationserverAPI.php

Order Status Updated

Order status is updated to Success, Failed, or Cancelled based on payment verification result.

Course Access Granted

Student automatically gets access to the purchased course upon successful payment.

Troubleshooting Guide

Solutions to common issues you might encounter

Payment Not Processing

Your payments are failing or not going through to SSLCommerz

1 Check Store ID and Password

Verify that your SSLCommerz Store ID and Password are entered correctly in Tutor LMS settings. Even a single character difference will cause payment failures.

2 Ensure Environment Matches

Your plugin environment (Sandbox or Live) must match your credentials. Using sandbox credentials in live mode or vice versa will cause authentication failures.

3 Verify IPN URL Configuration

Ensure your Instant Payment Notification (IPN) URL is correctly configured in SSLCommerz merchant panel. Without this, payments won't be verified.

4 Check SSL Certificate

Ensure your website has a valid SSL certificate (HTTPS). SSLCommerz requires secure connections for all communication.

Validation Failed

Transaction validation errors are preventing payment processing

1 Check IPN URL Accessibility

Verify that your IPN URL is publicly accessible. Use an online tool to check if the URL responds correctly from SSLCommerz servers.

2 Verify Webhook URL in Settings

Double-check that the webhook_url parameter in Tutor LMS settings points to your correct IPN endpoint.

3 Enable WP_DEBUG

Enable WP_DEBUG in your wp-config.php file to get detailed error messages that will help identify the exact validation issue.

4 Check Error Logs

Check your WordPress error logs and server logs for detailed error messages related to the validation failure.

Status Not Updating

Order status is not being updated after successful payment

1 Verify IPN Configuration

Confirm that IPN is properly enabled and configured in your SSLCommerz merchant account with the correct callback URL.

2 Check Order ID Parameter

Ensure the order ID is being correctly passed in the value_a parameter during checkout to properly identify which order is being paid.

3 Ensure Hash Verification Works

Hash verification must pass for the IPN to be processed. Check that your Store Password is correct and being used for hash generation.

4 Check Webhook Response

Test the webhook manually and check the browser console or server logs to see what response is being sent back to SSLCommerz.
